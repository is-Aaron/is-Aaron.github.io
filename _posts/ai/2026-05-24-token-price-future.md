> **核心观点**：判断 token 价格，不能只看最前沿模型的挂牌价，而要看“完成某类任务所需的最低 token 质量”。大多数真实问题并不需要最先进模型，只需要达到任务的够用线。只要算力发展、推理工程和模型效率的提升速度高于大多数任务对 token 质量的提升需求，满足同等任务质量的 token 成本就会越来越低。

我最近想到一个很朴素的类比：普通数据线、低端小尺寸显示器这类基础硬件，一旦进入成熟供应链，价格就会被压到很低的区间。这不是因为铜、塑料、面板、驱动板突然没有成本了，而是因为生产规模、工艺良率、渠道竞争和产品标准化一起把价格打了下来。

大模型 token 也会经历类似过程，但这个类比还不是核心。更核心的问题是：**大多数任务需要的不是最高质量，而是够用质量**。

## 一、先把问题问准：不是 token 单价，而是任务有效成本

讨论 token 价格时，很容易把三个问题混在一起：

| 问题 | 关注点 | 容易误判的地方 |
|------|--------|----------------|
| 每百万 token 挂牌价 | 某个模型输入、输出、缓存 token 的 API 单价 | 会把前沿模型价格误当成所有任务价格 |
| 同等能力价格 | 达到某个 benchmark 或能力门槛的最低成本 | 更适合观察技术进步带来的降价 |
| 任务有效成本 | 完成一次真实任务并达到验收标准的总成本 | 最接近产品和工程决策 |

我更关心第三个：任务有效成本。

一个 AI 客服回复不是为了消耗 token，而是为了解决一次咨询；一次代码审查不是为了生成几千个字，而是为了发现真实风险；一次日志分析不是为了让模型“读完日志”，而是为了定位根因。用户最终购买的不是 token 本身，而是“解决问题的能力”。

所以本文说“token 价格会越来越低”，不是说旗舰模型的最高规格 token 会很快变成白菜价，而是说：

**完成同等任务、达到同等质量要求所需的 token 成本会越来越低。**

这里还要注意一个细节：不同模型、不同服务商的 token 并不完全可比。不同 tokenizer 会让同一段文本切出不同数量的 token。Anthropic 的价格文档就提醒，Claude Opus 4.7 使用了新的 tokenizer，同一段固定文本相比旧模型可能多出最多 35% 的 token。因此，跨模型比较不能只看“每百万 token 几美元”，还要看同一任务实际消耗多少 token、是否成功、是否需要人工返工。

一个更接近真实业务的口径是：

**任务有效成本 = 输入成本 + 输出成本 + 缓存成本 + 工具成本 + 失败重试成本 + 人工复核成本。**

这个口径下，便宜模型如果经常失败，并不便宜；贵模型如果一次完成关键任务，也未必贵。真正会长期下降的，是“达到同等任务质量线的最低成本”。

## 二、可以从哪些角度切入

围绕 token 价格，可以从几个角度分析。

| 切入角度 | 关键问题 | 对结论的影响 |
|----------|----------|--------------|
| 任务质量需求 | 这个任务到底需要多强的 token？ | 多数任务有够用线，不需要前沿模型 |
| 边际效用递减 | 模型更强之后，任务结果还能提升多少？ | 过线之后，用户不愿为过剩质量付高价 |
| 算力供给 | 同样的钱可以买到多少推理算力？ | 单位算力成本下降会压低旧能力价格 |
| 模型效率 | 同样算力能产生多少可用智能？ | 小模型、蒸馏、MoE、量化会让旧能力下沉 |
| 推理工程 | 同样 GPU 能吐出多少有效 token？ | 批处理、KV Cache、PagedAttention、FlashAttention、投机解码会提高吞吐 |
| 缓存复用 | 重复上下文是否要反复计算？ | 稳定前缀、项目规则、工具说明可以摊薄成本 |
| 延迟容忍度 | 任务是否必须实时返回？ | Batch / 异步 / 低优先级通道通常能换取更低价格 |
| SLA 和合规 | 是否需要低延迟、专属容量、数据驻留、审计？ | 生产级服务可能保留溢价 |
| 开源和本地部署 | 任务能否由开源或本地模型够用完成？ | 会形成价格锚，但要计算运维和硬件成本 |
| 使用量变化 | 单价下降后会不会用得更多？ | 单次任务更便宜，但总 token 消耗可能上升 |

这几个角度合起来，比单纯讨论“模型公司会不会降价”更可靠。价格不是靠愿望下降，而是靠供给能力、工程效率、市场竞争和任务需求之间的关系下降。

## 三、需求侧：大多数任务有“够用线”

所谓“token 质量”，不是官方计费概念，而是一个分析用法：它指模型在特定任务上体现出的准确性、稳定性、推理能力和可用性。

不同任务需要的 token 质量差别很大。

| 任务 | 质量线 | 是否需要前沿模型 |
|------|--------|------------------|
| 文本分类 | 标签稳定、错误率可接受 | 通常不需要 |
| 信息抽取 | 字段完整、格式正确、少漏少编 | 通常不需要 |
| 摘要 | 覆盖重点、不改变事实 | 多数情况不需要 |
| 格式转换 | 结构正确、能被程序解析 | 不需要 |
| 普通客服 | 能回答常见问题、不乱承诺 | 多数情况不需要 |
| 局部代码修改 | 理解局部上下文、通过测试 | 视复杂度而定 |
| 架构决策 | 理解约束、权衡长期影响 | 可能需要强模型 |
| 高风险合规判断 | 准确、可审计、低幻觉 | 可能需要强模型和人工复核 |

这张表说明一个核心事实：**任务质量需求不是无限上升的**。

一个分类任务达到 98% 准确率后，继续把准确率从 98% 提到 98.5% 当然有价值，但未必值得付十倍价格。一个摘要任务能够稳定覆盖重点后，继续换更强模型可能只是让文风更漂亮，而不是让业务结果发生本质变化。一个普通客服机器人解决常见问题后，剩下的复杂问题可以转人工或升级到强模型。

这就是“够用线”。

大多数任务不需要最先进模型，它们需要的是：稳定、便宜、可评估、可回退。只要便宜模型跨过够用线，前沿模型的额外能力就会变成过剩质量。

## 四、供给侧：算力和模型效率在把旧能力推便宜

如果需求侧有够用线，那么供给侧只要继续进步，就会不断把更多便宜 token 推过这条线。

供给侧主要有三类进步。

第一是算力供给。GPU、专用推理芯片、内存带宽、网络互联、数据中心调度都会继续优化。短期内可能有供应紧张、电力、出口管制、资本开支等波动，但长期看，单位推理算力的获得方式会越来越成熟。

第二是模型效率。小模型、蒸馏模型、专用模型、MoE、量化模型会把很多任务从旗舰模型里剥离出来。Switch Transformer 这类 MoE 思路的核心就是稀疏激活：模型可以拥有很多参数，但每次计算只激活一部分专家，从而改变“参数规模”和“单次计算成本”的关系。蒸馏和专用模型则更直接：把通用大模型的某些能力压到更小模型里。

第三是推理工程。大模型推理不是简单“跑一下模型”。NVIDIA 对 LLM 推理的介绍里把推理分成 prefill 和 decode：prefill 阶段可以高度并行处理输入，decode 阶段则自回归地逐个生成输出 token。KV Cache 能减少重复计算，但会带来显存压力。PagedAttention 通过更高效地管理 KV Cache 降低内存浪费，vLLM 论文中报告了 2-4 倍吞吐提升。FlashAttention 通过减少 GPU HBM 与片上 SRAM 之间的读写，提高注意力计算效率。投机解码 / draft-and-verify 则试图一次草拟多个 token，再由目标模型验证，以减少串行生成的瓶颈。

这些技术方向的共同结果是：**同样硬件可以产出更多有效 token，同样任务可以用更小模型完成，同样上下文可以被更好复用**。

所以 token 降价并不是一个抽象判断，它有明确的工程路径。

## 五、价格下降已经发生过

这个判断不是纯想象，已经有数据支持。

Stanford HAI 的 2025 AI Index 提到，一个达到 GPT-3.5 水平的模型查询成本，从 2022 年 11 月每百万 token 20 美元，下降到 2024 年 10 月每百万 token 0.07 美元，约 18 个月下降超过 280 倍。报告还提到，不同任务上的大模型推理价格每年下降幅度可能从 9 倍到 900 倍不等。

这组数据说明的不是“最强模型变便宜了多少”，而是：**达到某个旧能力水平所需要付出的价格，下降得非常快**。

同一份 AI Index 还给了另一个角度：2022 年，在 MMLU 上超过 60% 分数的最小模型是 540B 参数的 PaLM；到 2024 年，3.8B 参数的 Phi-3-mini 就达到了类似门槛，参数规模减少了 142 倍。这并不意味着小模型全面替代大模型，而是说明很多能力会从前沿模型下沉到更小、更便宜、更容易部署的模型里。

另一篇基于 Artificial Analysis 和 Epoch AI 数据的研究也得出类似方向：在知识、推理、数学和软件工程 benchmark 上，前沿模型达到同等表现水平的价格大约以每年 5 到 10 倍的速度下降。同时，这篇论文也提醒了反方向的事实：由于模型变大、推理需求增加，运行前沿模型本身的价格可能仍在上升。

这正好支持本文的判断：下降最快的是“达到某个能力水平的成本”，不是“最前沿能力的绝对价格”。

## 六、为什么现在看起来仍然贵

截至 2026 年 5 月，主流服务商的公开价格页里仍能看到明显的价格阶梯。

OpenAI 的 GPT-5.1 标准价格是每百万输入 token 1.25 美元、每百万输出 token 10 美元。GPT-5.5 这类更前沿模型在短上下文标准层级下是每百万输入 token 5 美元、输出 token 30 美元；超过 272K 输入 token 后，官方说明会对 standard、batch、flex 层级的整个 session 按 2 倍输入价格和 1.5 倍输出价格计费。

Anthropic 的 Claude Sonnet 4.6 是每百万输入 token 3 美元、输出 token 15 美元；Claude Opus 4.7 是每百万输入 token 5 美元、输出 token 25 美元。Anthropic 的定价页还把 prompt caching 拆成 cache write、cache hit / refresh 等不同价格，说明真实账单并不只是输入和输出两个数字。

Google Gemini 也类似，Gemini 3.1 Pro Preview 在不超过 200K prompt token 的标准层级下是每百万输入 token 2 美元、输出 token 12 美元；超过 200K prompt token 后，输入和输出价格都会上调。Gemini 的价格页也明确写着部分模型的输出价格包含 thinking tokens，所以推理型任务不能只估最终可见回答的长度。

这些价格说明四件事。

第一，输出 token 通常比输入 token 贵。因为输入阶段可以并行处理，而输出阶段要自回归地一个 token 一个 token 生成，占用 GPU 时间更长。

第二，强推理、旗舰模型以及部分服务商的长上下文请求仍然有明显溢价。越接近前沿能力，价格越不容易快速下降。

第三，同一家服务商内部已经有很清楚的价格分层。旗舰模型、性价比模型、小模型之间可以差出很多倍。这说明市场已经不是“一个统一的 token 价格”，而是“不同质量层级的 token 价格”。

第四，真实成本会被缓存、工具、搜索、代码执行、上下文长度、服务层级、数据驻留等因素改变。只看模型名和每百万 token 单价，会低估生产系统的复杂度。

所以现在看起来贵，并不推翻 token 会降价的判断。它只是说明：**前沿 token 仍然贵，但够用 token 正在不断变便宜**。

## 七、价格会形成分层，而不是统一归零

未来 token 市场大概率会分成几层。

最底层是便宜 token。它们用于摘要、分类、格式转换、轻量客服、普通问答、低风险自动化。价格会非常低，低到开发者不再为一次调用犹豫，就像今天不会特别计算一根普通数据线里的铜线成本。

中间层是性价比 token。它们会承担大多数严肃工作：代码修改、文档生成、数据分析、产品原型、业务问答、批量审查。这一层竞争最激烈，也最像今天的主流显示器市场：大家会比较能力、延迟、稳定性、上下文长度和价格。

最高层是前沿推理 token。它们负责复杂数学、架构决策、长周期 Agent、重要代码变更、高风险合规判断。这一层不会很快变成白菜价，因为它卖的不是普通文本生成，而是更接近“稀缺判断力”的计算。

此外还有几种溢价层。

低延迟会有溢价。实时语音、交互式编程助手、在线客服等场景不能等太久，所以会为响应速度付费。

稳定容量会有溢价。企业要保证高峰期可用，可能需要专属容量、优先队列或更高服务等级。

合规和数据驻留会有溢价。某些行业必须控制数据位置、审计链路、访问权限和保留策略，这些不是普通 token 单价能覆盖的。

工具和环境会有溢价。搜索、文件检索、代码执行、浏览器、容器、长期存储都可能带来额外费用。

所以，token 不会简单归零。更可能发生的是：**基础智能越来越便宜，前沿判断和生产级保障继续收费**。

## 八、单价下降后，总使用量可能上升

还有一个容易忽略的角度：token 单价下降，不等于总账单一定下降。

数据线便宜之后，我们不是只买一根，而是在办公室、卧室、车里、背包里到处放。显示器便宜之后，很多人不是只省钱，而是开始双屏、竖屏、便携屏、多设备扩展。

token 也一样。

当 token 足够便宜，很多过去舍不得做的事情会变成默认能力：

- 每份文档自动维护摘要；
- 每个 PR 自动审查；
- 每个用户都有长期偏好和记忆；
- 每次线上异常都有自动解释；
- 每次学习都有个性化练习；
- 每个业务系统都有自然语言查询入口；
- 每个 Agent 任务都包含规划、检索、执行、验证和复盘。

这意味着：**单次任务成本下降，但 token 总消耗可能上升**。

所以对个人用户来说，感受可能是“AI 越来越便宜”；对企业系统来说，更准确的感受可能是“单位任务成本下降，但 AI 被嵌入更多流程，总预算需要重新规划”。

这并不反驳降价趋势。它说明降价会改变使用习惯。

## 九、对开发者的启发

如果相信够用 token 会越来越便宜，AI 应用就不应该只围绕“省 token”设计，而应该围绕“任务质量线”设计。

第一，要给任务分级。

格式转换、摘要、分类、抽取，不应该默认走旗舰模型。局部代码修改、测试生成、普通文档，也可以先走性价比模型。只有架构判断、复杂推理、高风险变更，才需要最强模型介入。

第二，要建立质量评估。

不要只看模型名字，而要看任务完成率、错误率、人工返工率、延迟和成本。只要便宜模型能稳定过线，就没有必要为过剩质量付费。

第三，要做模型路由。

未来价格和能力变化很快，今天最贵的能力明天可能被便宜模型覆盖。应用层应该能切换模型，而不是把模型名写死在业务逻辑里。

第四，要记录真实成本。

至少记录每个用户、每个任务、每个模型用了多少输入、输出、缓存 token，是否调用工具，是否命中缓存，是否成功，是否需要人工返工。没有这些数据，就无法判断一个任务该不该从旗舰模型迁移到更便宜的模型。

第五，要重视缓存和复用。

项目规则、产品文档、代码库摘要、工具说明、用户长期偏好，都应该被结构化保存和复用。未来 token 会更便宜，但重复上下文仍然浪费。

第六，要把实时和异步分开。

用户正在等待的交互，需要低延迟；批量审查、离线摘要、历史日志归类、文档批处理，可以走 Batch、异步队列或低优先级通道。很多服务商都给 Batch 或低优先级处理提供折扣，这本质上是用时间换价格。

第七，要保留升级通道。

便宜模型先处理，失败、低置信度或高风险任务再升级到强模型。这比所有任务都走旗舰模型更合理，也比盲目使用便宜模型更稳。

未来好的 AI 系统，应该像一个调度器，而不是一个单模型入口：简单任务走便宜模型，重复上下文走缓存，批量任务走异步，关键判断才走强模型。

## 十、这套判断的边界

说 token 会越来越便宜，不等于说所有东西都会便宜。

第一，前沿能力可能继续变贵。更长上下文、更复杂推理、更强工具调用、更高可靠性都会增加成本。前沿模型卖的是“新能力”，不是旧能力。

第二，生产级保障会继续收费。低延迟、稳定容量、审计、安全、数据驻留、私有化部署、企业支持，都不是单纯 token 推理。

第三，任务质量线也会提高。今天普通客服能回答 FAQ 就够了，未来用户可能要求它理解订单、调用工具、处理异常、识别情绪。需求侧不是静止的，只是很多任务的质量线不会像前沿模型能力那样无限上升。

第四，便宜 token 可能带来更多浪费。Agent 如果没有边界，可能反复检索、重复思考、生成冗余内容，把单价下降吃掉。

第五，跨模型比较要小心。不同 tokenizer、不同上下文计费、不同 thinking token 计费、不同缓存策略，都会影响真实成本。

所以更严谨的结论不是“token 一定免费”，而是：

**在大量有明确质量线的任务上，达到同等效果的最低 token 成本会持续下降；但前沿推理、低延迟生产服务和高可靠保障会继续保留溢价。**

## 结语

token 价格下降的本质，不是模型公司发善心，也不是所有 AI 能力都会突然免费，而是供需关系变了。

需求侧，大多数任务只需要够用质量。供给侧，算力、模型效率和推理系统还在持续进步。市场侧，模型分层、开源模型、本地部署、Batch、缓存和服务商竞争都会把“够用智能”的价格往下压。

这就像数据线和显示器。最高端的线材和专业显示器仍然可以很贵，但“够用的一根线”“够用的一块屏幕”已经便宜到改变使用习惯。

未来 token 也会如此。最前沿 token 仍然昂贵，但够用 token 会越来越便宜。真正下降的，不只是每百万 token 的价格，而是人类获得智能协助的成本。

## 术语表

- **Token**：大模型处理文本的基本计量单位，可以近似理解为字、词或词片段。API 通常按输入 token 和输出 token 分开计费。
- **token 质量**：本文中的非官方概念，指某个模型生成或理解 token 时，在特定任务上体现出的准确性、稳定性、推理能力和可用性。
- **够用线**：某类任务可接受的最低质量标准。超过这条线后，继续提升模型能力的边际收益会下降。
- **任务有效成本**：完成一次真实任务并达到验收标准所需的总成本，包括 token、缓存、工具、失败重试和人工复核等成本。
- **输入 token**：用户发送给模型的内容，包括问题、上下文、文档、代码、工具说明等。
- **输出 token**：模型生成的内容。由于生成过程通常需要逐个 token 串行解码，输出 token 往往比输入 token 更贵。
- **thinking tokens**：部分推理模型内部用于推理的 token，具体是否可见、如何计费取决于服务商和模型。
- **推理成本**：模型在生成答案时消耗的计算、显存、带宽、电力、调度和服务成本。
- **Prompt caching**：提示词缓存。把重复出现的长上下文复用起来，减少重复计算和费用；具体收益取决于缓存命中、写入成本、有效期和服务商计费方式。
- **MoE**：Mixture of Experts，混合专家模型。通常对每个 token 或样本路由到部分专家参数，而不是激活全部专家，以降低计算量。
- **KV Cache**：Transformer 解码时缓存历史 token 的 Key/Value 向量，减少重复计算，但会消耗显存。
- **Agent**：能够拆解任务、调用工具、执行多步骤操作并根据结果继续决策的 AI 系统。
