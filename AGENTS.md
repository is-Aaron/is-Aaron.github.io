# Project Agents Guide

本项目是一个基于 Jekyll Chirpy 的个人博客。Agent 在本项目内处理内容时，应优先遵循下面的项目级约定。

## 博客内容结构

- 普通文章放在 `_posts/<category>/` 目录下。
- 语录稿统一放在 `_posts/quotes/` 目录下。
- 语录页 `_tabs/quotes.md` 会按 `categories: [Quotes]` 自动收集文章，不需要手动维护列表。

## 语录稿写法

新增语录时，使用独立 Markdown 文章：

```markdown
---
title: "标题"
date: YYYY-MM-DD 09:00:00 +0800
categories: [Quotes]
tags: [quote]
---

> 语录原文。

## 随记

简短记录。
```

约定：

- 文件名使用 `_posts/quotes/YYYY-MM-DD-english-slug.md`。
- `categories` 固定为 `[Quotes]`。
- 默认 `tags` 使用 `[quote]`，除非用户明确给出更具体的标签。
- 语录原文必须尽量保留用户提供的文字、标点和语气，不擅自改写。
- 如果用户只说“记录语录”，先只写语录本身，不主动添加长篇解释。
- 如果用户要求补充想法，标题使用 `## 随记`，不要写成 `## 阅后感`。

## 随记风格

- 随记应像私人记录，不写成正式读后感、赏析或文章评论。
- 文字保持克制、短句、自然，可以是一到三小段。
- 不需要列点拆解含义，除非用户明确要求。
- 不要过度解释作者意图，也不要写成概念分析。
- 可以保留一点情绪和画面感，但不要太满。

## 当前语录稿示例

```markdown
---
title: "一棵树，我在这里等你"
date: 2026-05-22 09:00:00 +0800
categories: [Quotes]
tags: [quote]
---

> 一棵树，有人问是什么意思，我说：我在这里等你。

## 随记

像一棵树那样等你。

不靠近，也不催促。只是把根扎在这里，把枝叶伸向风里，让经过的人知道：我没有走。
```
